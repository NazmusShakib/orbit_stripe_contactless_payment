Quick start (TL;DR)

Phase 1 — Dev / Test Mode | Production

Integrates Stripe Terminal for contactless in-person card payments directly inside Odoo 16. Phase 1 uses Stripe's test/simulation environment — no real hardware required.

Features

  • Admin settings for Stripe API credentials (test mode)
  • Create Stripe PaymentIntents from Odoo backend
  • Simulate contactless card-present payments
  • Full payment audit log with chatter / activity tracking
  • Access control: User and Manager groups
  • Ready to extend for real Stripe Terminal hardware (Phase 2)

Payment Flow (3 Steps)

Step 1: Create Payment Intent — sends amount to Stripe, gets PaymentIntent ID
Step 2: Simulate Card Tap — triggers Stripe's simulation endpoint
Step 3: Confirm Payment — retrieves final status and Charge ID from Stripe

Configuration

Go to Settings → Stripe Terminal and enter:

  • Stripe Secret Key (sk_test_...)
  • Stripe Publishable Key (pk_test_...)
  • Terminal Location ID (tml_...)
  • Simulated Reader ID (tmr_...)

Requirements

  • Odoo 16
  • Python package: stripe (install via pip)
  • Stripe account with Terminal enabled
